流体赛道baseline解读

  • 该赛题给出了 101 个时间切片上的 16X16 低分辨率的数据，并给出了物理方程（PDE）；要求推测 512X512 高分辨率的数据。本质上是通过边界条件（数据点）解方程。特点是边界条件是 ill-posed，很难通过传统数值方法求解，是一个反问题（inverse problem）。

  • 针对这类问题的求解思路，包括但不限于 Physics-informed Neural Network（PINN） 方法。本baseline 展示了一个采用 PINN 方法解这个问题的思路。

  • PINN 方法的主要思想是利用神经网络可以自动计算偏微分的特性，在物理场中布置 n 个采样点 （Ppde​），计算这些采样点的预测物理量带入方程后的残差（epde​）；同时在已知真实数据的 16X16 个点上 （Pdata​），计算这些采样点的预测数据与真实数据之间的误差（edata​）。将神经网络的损失函数定义为 e=epde​+λedata​，其中 λ 是权重因子。当神经网络的 loss 被优化到足够小时，即认为神经网络在已知数据点上的预测值很接近真实值，在采样点上，预测值符合物理方程的规律。即得到了问题的解。

  • 注意，采样点（Ppde​）的位置选取可以是随意的，数量也没有约束。可以和（Pdata​）的位置重合，也可以在空间上任意分布。这些采样点的数量和位置的设计，是影响 PINN 方法最终效果的一个因素。

  • 模型中的权重因子 λ 是 PINN 优化过程中数据的 error 和 PDE 的 error 之间的权重。因为 epde​ 和 Pdata​ 计算出来的量级可能不一样，所以 λ 的选择需要尝试。有一些在训练 PINN 的过程中动态调整 λ 的策略可以提升 PINN 最终结果的准确度。 λ 的设计是影响 PINN 方法最终效果的一个影响因素。

  • PINN 方法已有一些开源的package，以下是两个例子： https://github.com/lululxvi/deepxde https://github.com/jpzxshi/learner

  • data_16.h5文件，是已知的 16X16 的低分辨率数据。h5文件中有 101 个 dataset，一个 dataset 是一个时间切片，dataset 的名称 ['000.0'，'000.3'，'000.6',,,'030.0']是时间戳。一个 dataset 是一个 256X5 的数组，5 列的信息分别是 [x坐标，y坐标，x方向的速度，y方向的速度，压力]。

  • Prediction.h5文件，是 PINN 方法预测的高分辨率数据，也是选手提交评测的文件。和Data_16.h5 相同格式，但每个 dataset 扩展为 262144x5。请保持时间戳的格式和每列信息的格式一致。数据点请按以下顺序排列：

    d = 2*np.pi/512
x = np.linspace(-np.pi, np.pi-d, 512)
y = np.linspace(-np.pi, np.pi-d, 512)
coordinates = [[xi, yi] for xi in x for yi in y]

  • 具体模块和变量的解释见example的主文件demo_pinn.py，这个文件展示了用PINN的主要流程。该文件只对一个较小的时间区间进行了求解，得到了相对不错的结果，供选手参考。
